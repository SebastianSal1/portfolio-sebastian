## Context

Backend developer internship at Ikonsultan Inovatama (Aug–Sep 2024). Product: web application helping MSMEs access financial services. Screenshots show a **training/demo** banking UI used to practice transaction flows — not the production MSME product UI.

## Problem

MSME users needed reliable financial transaction capabilities (withdrawal, transfer, deposit, payment) and readable bank statements — delivered inside a short internship window with a cross-functional team.

## Role

Backend developer intern. Built and iterated transaction features for the product team; coordinated via Git and Notion. Not sole system architect.

## Approach

- Developed **4 core transaction modules**: withdrawal, transfer, deposit, payment
- Built **dynamic bank statements** for account visibility
- Shipped **10+ feature updates in 4 weeks** with the product team

## Constraint

Employer systems and customer data are confidential — no public repo or live demo claimed here. Stack details only where already stated in CV/timeline.

## Result

Transaction modules and statement features delivered into the team’s release cadence. Software evidence sits as **implementation depth** under a strategy-and-analysis primary identity — not as a pure software-engineer pitch.
